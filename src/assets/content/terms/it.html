2. Descrizione del Servizio

Bam! è una piattaforma di file sharing peer-to-peer (P2P) che consente il trasferimento diretto di file tra browser tramite tecnologia WebRTC. I file non vengono mai caricati su server cloud: il trasferimento avviene esclusivamente in modalità diretta tra i dispositivi degli utenti.

Il servizio è disponibile come applicazione web accessibile da browser e come applicazione desktop tramite Electron, per i sistemi operativi Windows, macOS e Linux.

3. Account e Registrazione

Bam! offre tre livelli di utilizzo:

  • Anonimo (Guest) — Nessuna registrazione richiesta. L'utente può utilizzare il servizio immediatamente con funzionalità di base.
  • Pro Free — Richiede registrazione gratuita tramite email o autenticazione OAuth (Google, GitHub). Offre funzionalità aggiuntive come il ripristino del download.
  • Pro Premium — Richiede un abbonamento a pagamento. Include tutte le funzionalità avanzate senza limitazioni e senza pubblicità.

L'utente è responsabile della sicurezza delle proprie credenziali di accesso e di tutte le attività svolte tramite il proprio account.

4. Limitazioni d'Uso

I limiti di utilizzo variano in base al livello di account:

  • Anonimo e Pro Free: massimo 5 trasferimenti di file al giorno.
  • Pro Premium: trasferimenti di file illimitati.
  • Dimensione file: illimitata per tutti i livelli di account.

Gli utenti Anonimo e Pro Free visualizzeranno annunci pubblicitari durante l'utilizzo del servizio. Gli utenti Pro Premium usufruiscono di un'esperienza completamente priva di pubblicità.

7. Crittografia e Sicurezza

Tutti i trasferimenti su Bam! sono protetti da crittografia end-to-end (E2EE) nativa, garantita dal protocollo DTLS integrato in WebRTC. Questo significa che i dati in transito sono cifrati e non possono essere intercettati da terze parti.

  • Crittografia AES-GCM opzionale: disponibile per gli utenti Pro Premium, consente di aggiungere un ulteriore livello di protezione tramite password con derivazione della chiave via PBKDF2.
  • Verifica integrità: hash SHA-256 per garantire che i file ricevuti siano identici a quelli inviati.

Bam! non ha la possibilità tecnica di accedere al contenuto dei file trasferiti tra gli utenti. La natura P2P del servizio garantisce che nessun dato transiti sui nostri server.
